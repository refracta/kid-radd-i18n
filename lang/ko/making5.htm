.
솔직히 말씀드리면, 저는 자바스크립트를 잘하지 못합니다. HTML은 꽤 오래 독학했지만 자바스크립트를 제대로 파고들 만큼의 인내심이나 흥미를 아직 충분히 끌어올리지는 못했습니다. 더 나은 게임을 만들고 싶으니 언젠가는 제대로 공부하겠지만요...

보통 자바스크립트가 꼭 필요할 때는 퍼블릭 도메인 스크립트 라이브러리에서 코드를 찾아 제 목적에 맞게 조심스럽게 고쳐서 사용했습니다. 제 코믹 뷰어도 정확히 그렇게 만들었습니다. htmlgoodies.com의 이미지 전환 스크립트를 가져와 몇 군데를 수정했습니다. 먼저 링크를 메인 이미지와 분리된 내부 링크로 바꿨습니다. (일반적인 이미지 전환 링크는 페이지 내 다른 그림이 아니라, 링크 안쪽 이미지 자체를 바꾸는 경우가 많습니다.) 그리고 실행 트리거를 OnMouseOver에서 OnClick으로 바꿨습니다.

이제 스트립 1번 패널의 내부 링크를 클릭하면, 2번 패널로 이동하는 동시에 그 패널의 애니메이션 GIF가 바로 시작됩니다. 예시는 아래와 같습니다:

여기 클릭!


다음 패널



예시 코드입니다. 아래 부분은 문서의 "head" 영역에 들어갑니다:

<SCRIPT LANGUAGE="JavaScript">
<!-- hide from non-javascript browsers
Image1= new Image(63,96)
Image1.src = "raddrock.gif"


function SwapOut() {
document.imageflip.src = Image1.src;
}

// - stop hiding -->
</SCRIPT>


 그리고 아래는 실제 테이블 부분으로, 내부 링크를 눌렀을 때 두 번째 테이블의 이미지를 "교체"하는 코드입니다:

<center>
<table cellspacing="6" cellpadding="6" border="0">

</td></tr>

</td></tr></table>

 또 보통은 아직 보이지 않는 이미지(여기서는 래드의 에어기타 장면)를 문서 어딘가에 1x1 크기로 숨겨 둡니다. 위 자바스크립트가 이미지를 미리 로드하므로 원칙적으로는 불필요해야 하지만, 일부 브라우저에서는 기대처럼 동작하지 않는 경우가 있습니다. 일종의 미신처럼 느껴질 수도 있지만, 숨김 이미지를 넣어 두는 건 손해가 없고 독자가 링크를 눌렀을 때 애니메이션이 즉시 뜨도록 하는 보험 역할을 합니다.

더 효율적인 코드를 작성하실 수 있는 진짜 HTML 고수분들께는 다시 한 번 양해를 구하고, 반대로 제가 설명을 복잡하게 만들었다면 그 점도 죄송합니다. 테이블, 프레임, 자바스크립트 등 기초 튜토리얼은 여전히 HTML Goodies를 추천드립니다. 제가 여기까지 오는 데 큰 도움이 된 사이트입니다.
